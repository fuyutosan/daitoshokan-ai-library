# 📚 大図書館 — AIが自分を覚えてくれない問題、これで終わり

ChatGPTやClaudeに、毎回「私はこういう人間で、こういうプロジェクトをやっていて…」と説明し直すのに疲れていませんか？

**大図書館**は、AIとの作業コンテキスト（あなたのこと・プロジェクトのこと・過去の判断）を「目録→棚→日誌」という3層のシンプルなMarkdownファイルで貯めておく仕組みです。プログラミング不要、コピペと箇条書きだけで作れます。

---

## 🙋 こんな悩みありませんか？

- **毎回説明し直すのが大変** — 新しい会話を始めるたびに、自分のこと・プロジェクトのことを一から話している
- **長く話しているとAIがだんだん馬鹿になる** — 会話が長くなるほど、AIが前の方の指示を忘れたり、変な回答をし始める（これは「コンテキスト劣化」というAI側の性質で、あなたのせいではありません）
- **前回何を話したか忘れられる** — セッション（会話）をまたぐと、AIの記憶はリセットされてしまう

大図書館は、この3つを**AIが自分で読み書きする「外部メモ」**を作ることで解決します。あなたが覚えておく必要はありません。AIに「目録を見て」と言うだけで、必要なメモを自分で探して読みに行ってくれます。

---

## 🏗 仕組み（図解）

```
┌─────────────────────────────────────────┐
│  目録.md  ← 毎回これ「だけ」読む案内板     │
│  「棚が4つあります。今回は "仕事" の棚を  │
│   見てください」                          │
└───────────────┬───────────────────────────┘
                │ 関係する棚だけ開く
                ▼
┌─────────────────────────────────────────┐
│  棚/仕事.md, 棚/趣味・創作.md ...          │
│  ← テーマごとに整理された「知識」          │
│  （プロジェクトの経緯、決めたこと、好み）  │
└───────────────┬───────────────────────────┘
                │ 週1回、AIが整理して転記
                ▲
┌─────────────────────────────────────────┐
│  日誌/2026-07.md                          │
│  ← 作業が一区切りつくたびに5行だけ記録     │
│  「今日は何をした・どこに保存した・何を    │
│   学んだ」の生ログ                        │
└───────────────┬───────────────────────────┘
                │ 年に1回、AIが1年分を凝縮
                ▼
┌─────────────────────────────────────────┐
│  年鑑/2026.md（任意・育ってきたら）        │
│  ← 1年分をたった1ページに圧縮した年表      │
└─────────────────────────────────────────┘
```

**ポイント：情報は下から上に「圧縮」されていきます。**
日々の細かい記録（日誌）→ テーマごとの知識（棚）→ 1年分の要約（年鑑）。
新しい会話では一番上の「目録」だけ見れば、関係する棚に案内してもらえるので、AIが読み込む量は最小限で済みます（これがコスト削減にも、AIの回答の質を保つことにも直結します）。

## 🪄 いちばん簡単な導入 — このプロンプトを貼るだけ

手動セットアップが面倒な人は、Claude Codeにこの文をそのままコピペしてください。ダウンロードから棚のカスタマイズまで、AIが全部やってくれます。

```
https://github.com/fuyutosan/daitoshokan-ai-library にある「大図書館」を導入してください。

1. このリポジトリを一時フォルダに取得する（git clone か、ファイルを個別ダウンロード）
2. その中の library/ フォルダを、今の作業フォルダの直下にコピーする
   （すでに library/ がある場合は上書きせず、私に確認してください）
3. snippets/claude-md-snippet.md の内容を、作業フォルダの CLAUDE.md に追記する
   （CLAUDE.md が無ければ新規作成）
4. library/目録.md と library/運用ルール.md を読んで、仕組みを理解する
5. 私に「主にどんな仕事・プロジェクト・趣味に使うか」を3つ以内で質問して、
   棚の名前と目録を私に合わせてカスタマイズする
6. 最後に、何をどこに作ったかを一覧で報告する
```

CodexなどClaude Code以外のAIエージェントでも、プロンプト内の「CLAUDE.md」をそのツールの常時読み込みファイル名（Codexなら AGENTS.md）に読み替えれば同じように使えます。

手動でやりたい人・仕組みを理解してから入れたい人は、次の「5分セットアップ」へ。

## ⏱ 5分セットアップ

### ステップ1：フォルダをコピーする
この `daitoshokan-template` フォルダの中の `library/` フォルダを、あなたの作業フォルダの直下にコピーしてください。

```
あなたの作業フォルダ/
├── library/          ← ここにコピーする
│   ├── 目録.md
│   ├── 運用ルール.md
│   ├── 棚/
│   └── 日誌/
└── CLAUDE.md（Claude Codeを使っている場合）
```

Claude Code / Cowork以外（ChatGPTなど）を使っている場合は、単にこのフォルダをプロジェクトの参照ファイルとしてアップロードすればOKです。

### ステップ2：CLAUDE.mdに2行追記する（Claude Codeを使っている場合）
`snippets/claude-md-snippet.md` の中身をコピーして、あなたのプロジェクトの `CLAUDE.md`（無ければ新規作成）に貼り付けてください。これで「新しい仕事を始めるときは必ず目録を見る」というルールをAIに常時守らせられます。

ChatGPTや他のツールを使っている場合は、会話の最初に同じ内容を貼り付けるか、カスタム指示（Custom Instructions）欄に設定してください。

### ステップ3：簡単セットアッププロンプトをAIに貼る
`library/` をコピーしたら、AIとのチャットで以下をそのまま貼ってください。あなた専用の棚に整地してくれます。

```
library/ フォルダに大図書館システムを導入しました。
目録.md と 運用ルール.md、棚/ の中身を読んで、この仕組みを理解してください。
理解したら、私の状況（どんな仕事・プロジェクト・趣味に使うか）を3つ質問して、
棚の名前を私に合わせてカスタマイズしてください。
```

これで最低限のセットアップは完了です。あとは普段通り作業を進め、区切りがついたときに「日誌に記録して」と言うだけで運用が回り始めます。

## 📋 運用ルールの要約

詳しくは `library/運用ルール.md` を読んでください。ここでは3行で。

1. **仕事が一区切りついたら、日誌に5行だけ書く**（やったこと・成果物の場所・学び・振り分け先の棚）
2. **週1回、AIに「今週の日誌を棚に整理して」と頼む**（10分で終わる想定。全部読み返す必要はない）
3. **棚が育って情報が古くなったら、AIに「矛盾してる記述を直して」と頼む**（図書館は「書きっぱなし」ではなく「更新され続ける」ことで生きた資産になります）

## ❓ FAQ

**Q. プログラミングの知識は必要ですか？**
A. 不要です。すべて普通の日本語の文章（Markdownという、記号少なめの読みやすいテキスト形式）で書きます。

**Q. Claude Code以外（ChatGPT、Gemini等）でも使えますか？**
A. 使えます。仕組み自体はただのテキストファイルなので、会話の最初にlibraryフォルダの中身を貼り付ける・アップロードする形で応用できます。ただし「AIが自分で目録を見に行って必要な棚だけ読む」という自動化は、ファイルを自動で読めるツール（Claude Code、Cowork、カスタムGPTのファイル添付など）でこそ真価を発揮します。

**Q. 棚は4つ以上増やしてもいいですか？**
A. はい。`library/目録.md` の表に1行追記し、`library/棚/` に新しいファイルを追加するだけです。目録が長くなりすぎたら、関連する棚をサブフォルダにまとめることもできます（詳しくは運用ルール.md）。

**Q. 日誌を書き忘れたらどうなりますか？**
A. 特に問題ありません。書けるときに書く、くらいのゆるさで大丈夫です。ただし「同じ質問を何度もAIにしている」と感じたら、書く頻度を増やすサインです。

**Q. 家族や仕事仲間と共有できますか？**
A. できます。ただしlibraryの中身には個人情報や機密情報を書きがちなので、共有・公開する前には中身を必ず確認してください。

**Q. 有料版はありますか？**
A. この無料版だけでも十分に運用できます。「セットアップをAIと1個のプロンプトで自動化したい」「運用の実例をもっと見たい」という方向けの発展版は、別途ご案内する場合があります。
