# 図書館の運用ルール（書く人＝AI、読む人＝AI）

## 1. 日誌の書き方（作業の区切りごとに1回）
仕事が一区切りついたら `日誌/YYYY-MM.md` の末尾に以下の型で追記する。**5行以内厳守**。

```
## 2026-07-10｜〇〇の作業
- やったこと: 何をしたか一言で
- 成果物: ファイルの場所やURL
- 学び: 次に活かせる気づき
- 棚: 仕事
```

- 「棚:」に振り分け先を書いておくと週次整理が速い
- 生ログ（会話全文やツール出力）は書かない。要点だけを短く
- 雑談だけのセッション・5行にする価値がない作業は書かなくてよい

## 2. 棚の本の書き方
- 1エントリ＝見出し＋数行。「いつ・何をした・どこにあるか・何を学んだか」
- **正本（一番詳しい原本）があるものはコピーせずリンクする**（例:「詳しい手順はCLAUDE.md参照」と1行だけ書く）
- ルールに昇格すべき学び（同じミスを2回した、等）は棚ではなく絶対ルール側（CLAUDE.mdやカスタム指示）へ移す

## 3. 週1の整理ルーチン（10分想定）
毎週1回、以下だけをやる。**全巻の読み返しはしない**（情報が増えるほど破綻するため）。
1. 今週分の日誌エントリを読む
2. 各エントリを該当する棚の本に転記・統合する（既存エントリと重複したら統合する）
3. 棚を読んでいて**今の事実と矛盾する古い記述を見つけたら直す・消す**（改訂）
4. 棚が300行を超えたら分冊する → 目録も更新する

## 4. 長期成長設計（何年も生きている図書館にするために）
この図書館は長く育て続ける前提で設計する。

- **時間の圧縮階層**: 日誌（日単位・生の記録）→ 棚（テーマ単位・整理された知識）→ 年鑑（年単位・1年を1ページに凝縮）。古い詳細は下の層に沈み、上の層だけ読めば全体がわかる
- **年1回の年鑑ルーチン**（運用に慣れてきたら導入）: 前年の日誌12ヶ月分と棚の変化を `年鑑/YYYY.md` に1ページで凝縮する。項目例: その年の大きな出来事／育ったプロジェクト／最大の学び3つ／数字（作った物の数など）。書き終えた年の日誌は読み返し対象から卒業する（消さずアーカイブとして残す）
- **目録は常に1枚**: 棚が増えても目録が1画面に収まらなくなったら、棚側を階層化（サブフォルダ化）して目録には親項目だけ載せる
- **ツール非依存**: すべてプレーンなMarkdownのみ。特定のアプリ・AIモデル・形式に依存する書き方をしない（将来どんなAIやツールでも読めるように）
- **移設可能**: パスやツール名に依存する記述は最小限にし、フォルダごと引っ越しても成立する構造を保つ

## 5. 検索できない図書館は死んだ図書館
- ファイル名・見出しは普段使う普通の言葉で書く（後から検索して探せるように）
- 固有名詞は初出でフルネーム＋場所（ファイルパスやURL）を書く（「あれ」「例の件」禁止）
- 日付は必ず絶対表記（2026-07-10）。「先週」「昨日」は禁止（後から読むと意味が分からなくなるため）
