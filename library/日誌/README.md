# 日誌の書き方

日誌は「今月あったことの生の記録」を置く場所。月ごとに `日誌/YYYY-MM.md` というファイルを作り、作業が一区切りつくたびに末尾へ追記していく。

## 書き方の型（5行以内厳守）

```
## 2026-07-10｜〇〇の作業
- やったこと: 何をしたか一言で
- 成果物: ファイルの場所やURL
- 学び: 次に活かせる気づき
- 棚: 仕事
```

- 「棚:」は、週次整理のときにどの棚へ転記するかのメモ。書いておくと後の整理が速い
- 生ログ（会話全文やエラーメッセージの全文など）は書かない。要点だけ
- 雑談だけの日・記録するほどでもない作業は書かなくてよい

## 記入例

```
## 2026-07-12｜大図書館テンプレートの作成
- やったこと: 汎用版テンプレートをGitHub公開用に整備した
- 成果物: daitoshokan-template/ フォルダ一式
- 学び: 固有名詞を抜いて一般化すると、README単体で完結する構成にする必要がある
- 棚: AI活用術

## 2026-07-15｜サイトAのデザイン確定
- やったこと: クライアントとデザイン案を確定させた
- 成果物: `projects/サイトA/デザイン確定版.pdf`
- 学び: 配色は最初に希望色を聞いておくと手戻りが減る
- 棚: 仕事
```

書き終わったファイルはこの `日誌/` フォルダに `YYYY-MM.md` の名前でそのまま保存していけばOK。過去の月のファイルを消す必要はない（見返したいときのために残しておく）。
