# 📚 大図書館 目録（ここだけ読めば棚がわかる案内板）

AIとの全作業のコンテキストを保管する図書館。
**新しい会話ではまずこの目録1枚だけを読む。** 仕事に関係する棚の本だけを開いて読み込む。

## 使い方（3行）
1. 新しい仕事を始めたら、この目録を見て関係する棚を選ぶ
2. その棚の本（`棚/◯◯.md`）を読んでから作業に入る
3. 仕事が一区切りついたら `日誌/` に5行だけ記録を残す（書き方は `運用ルール.md`）

## 棚の案内（まずはこの4つの例から。自分に合わせて名前を変えたり増やしたりしてOK）

| 棚 | こういう仕事のときに開く |
|---|---|
| [仕事](棚/仕事.md) | 本業・副業・案件・プロジェクトの経緯や決定事項 |
| [趣味・創作](棚/趣味・創作.md) | 個人的な創作・趣味の作品・遊びのプロジェクト |
| [AI活用術](棚/AI活用術.md) | AIの使い方の工夫・コスト管理・過去の失敗と対策 |
| [自分の好み・判断基準](棚/自分の好み・判断基準.md) | 企画提案・デザイン判断・優先度づけの前に開く。自分の価値観や好みをAIに参照させる棚 |

## 棚の増やし方
- 新しいテーマの作業が2〜3回続いたら、専用の棚を1つ作る（`棚/◯◯.md`を新規作成）
- この表に1行追記するだけでOK。目録は常に1枚に収まる分量を保つ
- 棚が300行を超えて読みにくくなったら、内容ごとに分割する（例:「仕事」→「仕事_案件A.md」）。分割したら目録も更新する

## 年鑑（育ってきたら追加。年単位の凝縮記録）
- 運用に慣れてきたら `年鑑/` フォルダを作り、年に1回、その年の日誌12ヶ月分を1ページに凝縮する
- 最初のうちは無くてよい（棚と日誌だけで十分回る）

## 日誌（時系列の生の記録）
- `日誌/YYYY-MM.md` ← 今月の記録。月ごとに1ファイル

## 他の書庫との役割分担（重複させない）
- **CLAUDE.md（またはカスタム指示）** ＝ 絶対ルール（毎回自動で読まれる）。ルール化された結論だけ置く
- **この図書館（library/）** ＝ 「何をやってきたか」の歴史と経験知。ルール未満だけど後で効く知識
